## Objective

Learn to use Git to save your work and track changes. This is non-negotiable for professional development.

## Key Concepts

- **Repository (Repo):** A project folder that Git is watching.
- **Commit:** A saved checkpoint of your code.
- **`.gitignore`:** A file that tells Git to ignore certain files (like secrets and temporary files).

## The Task

Initialize a Git repository and make your first commit.

## Steps

1.  **Create a Project Folder:** Make a folder called `api_wrapper_cli`.
2.  **Initialize Git:**
    - **Prompt:** "I'm in my project folder `api_wrapper_cli`. Show me the git command to initialize a new repository."
3.  **Create `.gitignore`:**
    - **Prompt:** "Create a `.gitignore` file for a Python project. It should ignore virtual environments, `.env` files, and common Python cache files."
4.  **First Commit:**
    - **Prompt:** "Show me the git commands to stage all files and make my first commit with the message 'Initial commit'."

## Vibe Check

- If Git says `fatal: not a git repository`, you're in the wrong folder or forgot to run `git init`.
- Don't worry about understanding every Git concept yet. For now, just know: `git add` â†’ `git commit` = saved checkpoint.

## Success Criteria

Running `git log` shows your first commit.
